# Семинар 4: Музей

## О проекте

Проект описывает простую объектно-ориентированную модель музея и показывает,
как в предметной области «музей» используется полиморфизм, а также основные
отношения между объектами: композиция, агрегация, ассоциация и делегирование.

Код сделан в учебном стиле:

- без лишних шаблонов и сложных паттернов;
- с понятными классами и методами;
- с разбиением на `include/` и `src/`;
- со сборкой через `CMake`.

## Что реализовано

- Абстрактный базовый класс `MuseumItem`
- Виртуальный деструктор в базовом классе
- Две чистые виртуальные функции:
  - `printInfo()`
  - `getRecommendedVisitMinutes()`
- Иерархия наследования:
  - `MuseumItem`
  - `Exhibit`
  - `Painting`
  - `Sculpture`
  - `Artifact`
- Хранение объектов разных типов в одном контейнере:
  - `std::vector<std::shared_ptr<MuseumItem>>`
- Полиморфный перебор коллекции через базовый интерфейс
- Делегированный поиск экспоната через `Catalog`
- Демонстрация `typeid` и `dynamic_cast` в оправданном месте

## Основные классы

- `MuseumItem` — абстрактный базовый класс музейного объекта
- `Exhibit` — общий базовый класс экспоната
- `Painting` — картина
- `Sculpture` — скульптура
- `Artifact` — исторический артефакт
- `Hall` — зал музея
- `Catalog` — каталог для поиска экспонатов
- `Guide` — гид, который хранит маршрут экскурсии
- `Visitor` — посетитель, связанный с гидом
- `Museum` — основной класс музея

## Объектные отношения

- Композиция: `Museum` содержит объекты `Hall`
- Агрегация: `Guide` хранит маршрут из указателей на `MuseumItem`
- Ассоциация: `Visitor` хранит указатель на `Guide`
- Делегирование: `Museum` передаёт поиск экспоната классу `Catalog`

## Как демонстрируется полиморфизм

Полиморфизм показан не формально, а через реальные действия программы:

- объекты `Painting`, `Sculpture` и `Artifact` хранятся в одном контейнере через `shared_ptr<MuseumItem>`;
- при переборе коллекции вызываются виртуальные методы через базовый интерфейс;
- для каждого объекта полиморфно вызываются:
  - `printInfo()`
  - `getRecommendedVisitMinutes()`
- отдельно показано использование `typeid` и `dynamic_cast`, когда нужно получить
  доступ к специфическим полям производного класса.

## Сборка и запуск

Сборка:

```bash
cmake -B build
cmake --build build
```

Запуск:

```bash
./build/museum_app
```

## Что показывает программа

После запуска программа выводит:

- информацию о музее;
- список залов;
- коллекцию экспонатов;
- полиморфный вывод через базовый класс;
- план осмотра;
- маршрут экскурсии;
- работу гида и посетителя;
- поиск экспоната через каталог;
- пример использования `RTTI` и `dynamic_cast`;
- итоговый блок с кратким выводом.
