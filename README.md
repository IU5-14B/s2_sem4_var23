# Семинар 4: Музей и полиморфизм

## Что реализовано

В проекте показаны все требуемые элементы задания:

- Абстрактный базовый интерфейс: `MuseumItem`
- Две чистые виртуальные функции: `printInfo()` и `getRecommendedVisitMinutes()`
- Иерархия наследования: `MuseumItem -> Exhibit -> Painting / Sculpture / Artifact`
- Полиморфный контейнер: `std::vector<std::shared_ptr<MuseumItem>>`
- Ассоциация: `Visitor` хранит указатель на `Guide`
- Агрегация: `Guide` хранит маршрут из объектов `MuseumItem*`
- Композиция: `Museum` содержит объекты `Hall`
- Делегирование: `Museum` делегирует поиск экспоната классу `Catalog`
- Демонстрация `dynamic_cast` и `typeid`

## Краткое описание классов

- `MuseumItem` — абстрактный интерфейс музейного объекта
- `Exhibit` — общий базовый класс экспоната
- `Painting` — картина
- `Sculpture` — скульптура
- `Artifact` — исторический артефакт
- `Hall` — зал музея
- `Catalog` — каталог для поиска экспонатов
- `Guide` — гид, который хранит маршрут экскурсии
- `Visitor` — посетитель музея
- `Museum` — основной класс музея

## Сборка и запуск

Для сборки и запуска используйте команду:

```bash
cmake -B build && cmake --build build && ./build/museum_app
```

## Что делает программа

После запуска программа:

- выводит список залов музея
- полиморфно выводит информацию о разных типах экспонатов
- показывает рекомендуемое время осмотра для каждого объекта
- демонстрирует маршрут гида
- показывает связь посетителя с гидом
- выполняет поиск экспоната через каталог
- определяет фактический тип объекта через `typeid`
- получает специфичные данные производных классов через `dynamic_cast`
